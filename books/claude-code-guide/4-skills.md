# スキル機能の活用

## CLAUDE.md — プロジェクトの知識ベース

CLAUDE.md は Claude Code の最も強力な機能の一つです。プロジェクトのルールや慣習を記述しておくと、Claude Code が常にそれを参照して行動します。

### CLAUDE.md の配置場所

CLAUDE.md は複数の場所に配置でき、それぞれスコープが異なります。

```
~/.claude/CLAUDE.md          # ユーザーレベル（全プロジェクト共通）
./CLAUDE.md                  # プロジェクトレベル（リポジトリ全体）
./src/CLAUDE.md              # ディレクトリレベル（特定ディレクトリ）
```

より具体的なスコープの設定が優先されます。

### 効果的な CLAUDE.md の書き方

CLAUDE.md には、プロジェクト固有のルールやコンテキストを記述します。

```markdown
# プロジェクト概要
TypeScript + Express のREST APIサーバー。PostgreSQL を使用。

# コーディング規約
- 関数は必ず戻り値の型を明記する
- エラーハンドリングは Result 型パターンを使う
- テストは vitest で書く

# ディレクトリ構成
src/
  routes/      — API エンドポイント
  services/    — ビジネスロジック
  models/      — データモデル
  utils/       — ユーティリティ

# コマンド
- テスト実行: npm test
- lint: npm run lint
- ビルド: npm run build

# 注意事項
- .env ファイルをコミットしないこと
- マイグレーションは src/migrations/ に配置
```

### ユーザーレベルの CLAUDE.md

個人の作業スタイルをグローバルに設定できます。

```markdown
# ~/.claude/CLAUDE.md

# コミットメッセージ
- 日本語で書く
- 1行目は50文字以内の要約
- Why（なぜこの変更か）を本文に書く

# コードスタイル
- コメントは日本語で書く
- 変数名・関数名は英語
```

## カスタムスキルの作成

スキルは、よく使うワークフローを再利用可能な形でまとめたものです。

### スキルファイルの構造

スキルは `.claude/skills/` ディレクトリに Markdown ファイルとして保存します。

```
.claude/
  skills/
    commit.md
    review.md
    test-and-fix.md
```

### スキルの例: コミットスキル

```markdown
<!-- .claude/skills/commit.md -->
---
name: commit
description: 変更内容を確認してコミットする
---

# コミットスキル

以下の手順でコミットを行ってください。

1. `git status` と `git diff` で変更内容を確認する
2. 変更の目的と影響を分析する
3. 以下の形式でコミットメッセージを作成する:
   - 1行目: 変更の要約（50文字以内）
   - 空行
   - 本文: なぜこの変更が必要か
4. ユーザーに確認を取ってからコミットする
```

### スキルの例: テスト＆修正スキル

```markdown
<!-- .claude/skills/test-and-fix.md -->
---
name: test-and-fix
description: テストを実行し、失敗したテストを修正する
---

# テスト＆修正スキル

1. `npm test` でテストを実行する
2. 失敗したテストがあれば:
   a. エラーメッセージと該当コードを読む
   b. 原因を分析する
   c. 修正を適用する
   d. テストを再実行して確認する
3. 全テストが通るまで繰り返す
```

### スキルの呼び出し

スラッシュコマンドでスキルを呼び出します。

```
> /commit
> /test-and-fix
```

## 実践的なスキル集

### コードレビュースキル

```markdown
<!-- .claude/skills/review.md -->
---
name: review
description: コード変更のレビューを行う
---

以下の観点で変更をレビューしてください:

1. **正確性**: ロジックに誤りはないか
2. **セキュリティ**: OWASP Top 10 に該当する脆弱性はないか
3. **テスト**: テストは十分か
4. **可読性**: コードは理解しやすいか
5. **パフォーマンス**: 明らかな問題はないか

問題があれば具体的な改善案を提示してください。
```

### ドキュメント生成スキル

```markdown
<!-- .claude/skills/docs.md -->
---
name: docs
description: API ドキュメントを生成する
---

指定されたソースコードから API ドキュメントを生成してください。

- 各エンドポイントのメソッド、パス、説明
- リクエスト/レスポンスの型定義
- 使用例（curl コマンド）
- エラーレスポンスの一覧
```

CLAUDE.md とカスタムスキルを組み合わせることで、Claude Code をプロジェクト専用のアシスタントとしてカスタマイズできます。

# 実践的な Tips

## よく使うパターン

### パターン1: 既存コードの理解から始める

新しいプロジェクトに参加した時や、馴染みのないコードを触る時に有効です。

```
> このプロジェクトの全体構造を説明して。
> 特にデータの流れに注目して。
```

```
> src/services/payment.ts の処理フローを
> シーケンス図風に説明して
```

### パターン2: エラーからの復旧

スタックトレースをそのまま貼り付けるだけで、原因調査と修正案の提示が得られます。

```
> このエラーを修正して:
>
> Error: ECONNREFUSED 127.0.0.1:5432
>   at TCPConnectWrap.afterConnect [as oncomplete]
```

Claude Code は関連する設定ファイル（データベース接続設定など）を探し、原因を特定します。

### パターン3: リファクタリング

安全にリファクタリングを進めるパターンです。

```
> src/utils/helpers.ts が大きくなりすぎている。
> 関連する関数ごとにファイルを分割して。
> インポートパスも全て更新して。
> テストが通ることを確認して。
```

### パターン4: 依存関係の更新

```
> package.json の依存関係で、
> メジャーバージョンが上がっているものを教えて。
> 破壊的変更があるか調べて。
```

## トラブルシューティング

### Claude Code が応答しない

```bash
# プロセスを確認
ps aux | grep claude

# 再起動
claude
```

### コンテキストが失われる

長い作業セッションでは、コンテキストウィンドウの制限に達することがあります。

**対策:**
- 大きなタスクは小さく分割する
- 定期的に新しいセッションを開始する
- CLAUDE.md にプロジェクトの重要情報をまとめておく

### 意図しないファイル変更

Claude Code が予期しないファイルを変更した場合の対処法です。

```bash
# 変更を確認
git diff

# 特定のファイルの変更を取り消し
git checkout -- path/to/file

# 全ての変更を確認してから選択的にステージング
git add -p
```

### 権限エラー

ツールの実行が拒否された場合は、権限設定を確認します。

```bash
# 現在の設定を確認
cat .claude/settings.json

# 必要に応じて許可を追加
```

## 効率化テクニック

### 1. テンプレートを用意する

よく作成するファイルのパターンを CLAUDE.md に記述しておきます。

```markdown
# CLAUDE.md に追記

## コンポーネントテンプレート
新しい React コンポーネントを作成する際は以下の構造に従う:
- Props の型定義をインターフェースとして定義
- デフォルトエクスポートを使用
- テストファイルを同じディレクトリに配置
```

### 2. エイリアスを設定する

頻繁に使うコマンドをシェルエイリアスにします。

```bash
# .zshrc に追加
alias cc="claude"
alias ccp="claude -p"
alias ccr="claude -p 'テストを実行して、失敗があれば修正して'"
```

### 3. Git フックと組み合わせる

コミット時に自動でコードレビューを実行するようにします。

```bash
#!/bin/bash
# .git/hooks/pre-commit
claude -p "staged の変更をレビューして。問題があれば exit 1 で報告して" \
  --output-format json
```

### 4. 複数のモデルを使い分ける

タスクの性質に応じてモデルを切り替えます。

```bash
# 簡単な質問には Haiku（高速）
claude --model claude-haiku-4-5 -p "この関数の戻り値の型は？"

# 設計相談には Opus（高性能）
claude --model claude-opus-4-6
```

### 5. 出力を活用する

Claude Code の出力を他のツールと連携させます。

```bash
# TODOリストを生成してIssueに登録
claude -p "コードベースのTODOを一覧にして" | gh issue create --body-file -

# テスト結果の分析レポートを生成
npm test 2>&1 | claude -p "テスト結果を分析して改善案を出して"
```

## セキュリティに関する注意

- API キーや認証情報を Claude Code に直接渡さない
- `.env` ファイルの内容を Claude Code に表示させない
- 権限設定で機密ファイルへのアクセスを制限する
- CI/CD での使用時はシークレット管理を徹底する

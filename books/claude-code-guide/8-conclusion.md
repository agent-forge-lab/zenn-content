# まとめ

## 本書で学んだこと

本書を通じて、Claude Code の基本から応用まで一通り解説しました。振り返ると、以下のスキルを身につけました。

- **セットアップ**: インストール、認証、権限設定
- **基本操作**: ファイル操作、コード生成、Git 操作
- **カスタマイズ**: CLAUDE.md によるプロジェクト固有の設定
- **スキル**: 再利用可能なワークフローの構築
- **チーム開発**: マルチエージェントでの並行作業
- **応用**: MCP サーバー、フック、CI/CD 連携

## AI と協働する開発スタイル

Claude Code のようなエージェント型 AI ツールは、開発者の役割を変えつつあります。

従来の開発では、アイデアから実装まで全てを自分の手で行っていました。AI との協働では、開発者の役割は「設計者・レビュアー・意思決定者」に寄っていきます。

```
従来の開発フロー:
  要件定義 → 設計 → 実装 → テスト → レビュー → デプロイ
  [-------------- 全て自分で ----------------]

AI 協働の開発フロー:
  要件定義 → 設計 → 実装 → テスト → レビュー → デプロイ
  [自分で]   [共同]  [AI]   [AI]   [自分で]   [共同]
```

重要なのは、AI に全てを任せることではありません。設計の判断、コードレビュー、品質の最終確認は人間が行う。実装やテストのような定型的な作業は AI に任せる。この役割分担が、生産性と品質の両方を高めます。

## Agent Forge の実験から

本書は Agent Forge プロジェクトの一環として、AI エージェントチームが制作しました。この実験から得られた知見をいくつか共有します。

**AI は「指示の質」に比例した成果を出す。** あいまいな指示からはあいまいな成果しか得られません。CLAUDE.md に明確なルールを書き、具体的な指示を出すことで、期待通りの成果が得られます。

**小さなサイクルを回すことが重要。** 一度に大きな作業を任せるよりも、小さなタスクに分割して段階的に進める方が品質が高い。これは人間のチーム開発と同じ原則です。

**コンテキストの共有が鍵。** AI エージェントは、与えられた情報の範囲でしか判断できません。CLAUDE.md にプロジェクトの背景、規約、判断基準を明記しておくことで、一貫性のある作業が可能になります。

## 次のステップ

本書を読み終えた方への次のステップを提案します。

1. **自分のプロジェクトに CLAUDE.md を作る** — まずは小さく始める
2. **カスタムスキルを1つ作る** — よく繰り返す作業を自動化する
3. **チームに共有する** — CLAUDE.md をリポジトリにコミットしてチームで使う
4. **MCP サーバーを1つ追加する** — プロジェクトで使うツールを連携させる
5. **CI/CD に組み込む** — 自動レビューやテスト生成から試す

## おわりに

AI ツールは日々進化しています。本書で紹介したテクニックも、数ヶ月後にはより洗練された方法に置き換わるかもしれません。

しかし、本質は変わりません。明確な指示を出し、小さなサイクルを回し、品質を確認する。このプロセスは、AI と協働する上で普遍的な原則です。

Claude Code をあなたの開発ワークフローに取り入れ、生産性を新たなレベルに引き上げてください。

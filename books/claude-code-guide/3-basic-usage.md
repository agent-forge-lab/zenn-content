# 基本的な使い方

## 対話の基本

Claude Code は対話形式で操作します。ターミナルで `claude` を起動し、自然言語で指示を出します。

```bash
claude

# 対話モードで指示を出す
> src/utils.ts の parseDate 関数にバリデーションを追加して
```

ワンショット（1回きりの指示）で使うこともできます。

```bash
# パイプで入力を渡す
echo "このエラーの原因を教えて" | claude

# -p フラグで直接指示
claude -p "src/index.ts のテストを書いて"
```

## ファイル操作

### ファイルの読み取り

Claude Code はプロジェクト内のファイルを自律的に読み取ります。「このファイルを見て」と指示するだけで、内容を理解し要約してくれます。

```
> src/auth/login.ts を読んで、認証フローを説明して
```

### ファイルの作成・編集

ファイルの作成や編集も自然言語で指示できます。

```
> src/utils/validation.ts に、メールアドレスのバリデーション関数を作成して

> package.json の scripts に "lint" コマンドを追加して
```

Claude Code は変更前に差分を表示し、承認を求めます。変更内容を確認してから適用できるので安心です。

### コードの生成

具体的な仕様を伝えると、テスト付きのコードを生成してくれます。

```
> Express で REST API のエンドポイントを作って。
> GET /api/users でユーザー一覧を返す。
> バリデーションとエラーハンドリングも含めて。
```

## コード理解と質問

### コードの説明

既存のコードベースの理解に活用できます。

```
> この関数が何をしているか、ステップごとに説明して

> src/database/ ディレクトリのアーキテクチャを解説して
```

### バグの調査

エラーメッセージやスタックトレースを渡して、原因を調査してもらえます。

```
> このエラーの原因を調べて:
> TypeError: Cannot read properties of undefined (reading 'map')
> at UserList (src/components/UserList.tsx:15)
```

Claude Code はスタックトレースをたどり、関連するファイルを読み、原因と修正案を提示します。

## Git 操作

### コミット

変更内容を理解した上で、適切なコミットメッセージを生成してくれます。

```
> 今の変更をコミットして
```

Claude Code は `git diff` を確認し、変更の意図を汲んだコミットメッセージを作成します。

### PR の作成

GitHub CLI (`gh`) がインストールされていれば、PR の作成も可能です。

```
> この変更で PR を作成して。レビュアーに変更内容が伝わるように説明を書いて
```

## テストの実行と修正

### テスト実行

```
> テストを実行して、失敗しているテストがあれば修正して
```

Claude Code はテストランナーのコマンドを実行し、失敗したテストの原因を分析し、修正案を提案します。

### テスト作成

```
> src/utils/string.ts のユニットテストを作成して。
> 境界値とエッジケースも含めて。
```

## 効果的な指示の出し方

Claude Code を効果的に使うためのポイントをまとめます。

### 具体的に指示する

```
# あいまい
> ログイン機能を改善して

# 具体的
> ログイン失敗時のエラーメッセージを、
> 「認証に失敗しました」から具体的な原因
> （パスワード不一致、アカウント未登録）に変更して
```

### コンテキストを与える

```
> このプロジェクトは Express + TypeScript で構築されたAPIサーバーです。
> PostgreSQL を使用しています。
> 新しいエンドポイント POST /api/orders を追加してください。
```

### 段階的に依頼する

大きなタスクは小さなステップに分割して依頼すると、品質が向上します。

```
# ステップ1
> まず、Order モデルのインターフェースを定義して

# ステップ2
> 次に、バリデーションロジックを作って

# ステップ3
> 最後に、エンドポイントとテストを作って
```

この段階的なアプローチにより、各ステップで品質を確認しながら進められます。

# チーム開発

## マルチエージェント開発とは

Claude Code のマルチエージェント機能を使うと、複数の AI エージェントが並行して作業できます。人間の開発チームのように、それぞれのエージェントが異なる役割を担い、タスクを分担して進めます。

本書自体がこの機能を使って作られています。Agent Forge チームでは以下の構成で開発を行っています。

```
team-lead       — タスク管理と全体調整
strategist      — 市場調査と戦略立案
builder         — 設計と実装
meta            — プロセス改善とメトリクス
```

## エージェントの起動と管理

### サブエージェントの起動

Claude Code 内からサブエージェントを起動できます。Task ツールを使い、独立したコンテキストで並行作業を行います。

```
> src/auth/ のリファクタリングと、
> src/api/ のテスト作成を並行して進めて
```

Claude Code は独立したタスクを識別し、並行実行できるものは同時に処理します。

### タスクの分割と管理

大きなタスクを複数のサブタスクに分割し、それぞれを独立して進められます。

```
> 以下のタスクを並行して進めて:
> 1. ユーザーモデルのバリデーション追加
> 2. API エンドポイントのエラーハンドリング改善
> 3. 既存テストの修正
```

## CLAUDE.md によるチーム規約

チーム開発では、CLAUDE.md がコーディング規約の共有に活躍します。

```markdown
# CLAUDE.md

## チーム規約

### ブランチ命名規則
- feature/<issue番号>-<概要>
- fix/<issue番号>-<概要>
- docs/<概要>

### コミットメッセージ
- Conventional Commits 形式を使用
- feat: 新機能
- fix: バグ修正
- docs: ドキュメント
- refactor: リファクタリング
- test: テスト

### PR ルール
- レビュー1人以上の承認が必要
- テストが通っていること
- lint エラーがないこと

### テスト
- 新機能には必ずテストを書く
- カバレッジは80%以上を維持
```

エージェントもこの規約に従って作業するため、人間の開発者とエージェントが混在するチームでも一貫性が保たれます。

## ワークツリーによる並行作業

Git Worktree を使うと、同じリポジトリで複数のブランチを同時に操作できます。

```bash
# ワークツリーモードで Claude Code を起動
claude

# Claude Code 内で
> worktree で新しいブランチを作って作業して
```

これにより、メインブランチの作業を中断せずに、別のタスクを並行して進められます。

## 実践例: 機能追加の並行開発

実際のチーム開発フローを見てみましょう。

### 1. タスクの計画

```
> ユーザー管理機能を追加したい。
> 必要なタスクを洗い出して、並行実行できるものを特定して。
```

Claude Code が以下のようにタスクを分割します。

```
タスク1: User モデルとマイグレーション作成
タスク2: CRUD の API エンドポイント実装 (タスク1に依存)
タスク3: バリデーションミドルウェア作成 (独立)
タスク4: テスト作成 (タスク2に依存)
```

### 2. 独立タスクの並行実行

タスク1とタスク3は独立しているため、並行して実行できます。

### 3. 依存タスクの順次実行

タスク1が完了したらタスク2、タスク2が完了したらタスク4と順に進みます。

### 4. 統合とレビュー

全タスクが完了したら、変更を統合しテストを実行します。

```
> 全てのテストを実行して、問題がないことを確認して。
> その後、PR を作成して。
```

## チーム開発のベストプラクティス

1. **CLAUDE.md を充実させる** — チームの規約を明文化する
2. **タスクは小さく** — 1つのタスクは30分以内で完了できるサイズに
3. **依存関係を明確に** — タスク間の依存関係を事前に整理する
4. **テストを自動化** — エージェントが自律的にテストを実行できるようにする
5. **振り返りを行う** — 定期的にプロセスを見直し改善する
